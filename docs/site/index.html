Current install path

Source install, no sudo by default.

The installer builds the harness, installs the user-facing command, and copies runtime prompts and model catalogs so the command works outside the repository checkout. 

git clone https://github.com/dennisonbertram/go-agent-harness.git
cd go-agent-harness
./scripts/install.sh --add-to-path

go-code              # TUI
go-code "fix tests"  # single-shot
go-code --server     # persistent daemon

Built for everyday development

A harness you can actually live inside.

Workspace-aware

Launch from any repository. Runs carry that project path through to the server so tools operate in the place you intended.

Readable tool flow

Tool calls stay visible while work is active, then collapse after the assistant reply so the conversation stays usable.

Model-flexible

Use direct providers or gateways with catalog-backed model metadata, pricing, reasoning effort, and last-selected model memory.

Streamed and inspectable

The HTTP event stream exposes assistant deltas, tool events, usage, context, approvals, and run lifecycle state.

Three entrypoints

One command for interactive and scripted work.

go-code Launch the terminal UI in the current project.
go-code "prompt" Run one prompt, stream output, then exit.
go-code --server Start the local harness daemon and leave it running.

Distribution ladder

From local installer to release-quality packaging.

  1. Source installer Current channel for daily use and early testers.
  2. GitHub Releases Per-platform archives containing binaries plus `share/go-code` assets.
  3. Homebrew tap Preferred polished macOS path once release archives exist.
  4. Single binary Long-term simplification: embed assets and make `go-code` the whole product surface.
