Start here

One command for interactive and scripted coding loops.

Install from source today. The script builds the daemon and terminal client, then installs the user-facing `go-code` command plus its runtime assets. 

git clone https://github.com/dennisonbertram/go-code.git
cd go-code
./scripts/install.sh --add-to-path

go-code                          # open the TUI
go-code "summarize this repo"     # run once, then exit
go-code --server                 # start the local API daemon

How it feels

Keep the loop visible while the agent works.

go-code is built around ordinary repository work: open a project, ask for a change, watch tool calls happen, steer when needed, and keep enough state to continue later.

01

Open from any repo

The command resolves the current project and sends that workspace to the local harness so file and shell tools stay grounded.

02

Observe every step

Runs stream assistant deltas, tool calls, tool results, approvals, token usage, costs, and terminal status as they happen.

03

Steer mid-run

Send new context into an active session, cancel a bad path, approve sensitive work, or continue from an existing conversation.

Runtime primitives

A harness you can adapt instead of work around.

Workspaces Local directories and git worktrees, with observable lifecycle events.
Providers Catalog-backed models, aliases, pricing metadata, gateways, and direct provider routing.
Controls Sandbox scope, approval policy, max steps, max cost, role models, and allowed tools per run.
Context Conversation storage, replay, compaction, project instructions, skills, MCP, and subagents.

API first

The terminal is a client. The stream is the product surface.

Start `harnessd`, create a run, and subscribe to Server-Sent Events. Integrations can inspect lifecycle state, tool activity, usage, approvals, summaries, todos, and conversation history.

POST /v1/runs GET /v1/runs/{id}/events POST /v1/runs/{id}/steer POST /v1/runs/{id}/approve GET /v1/models GET /v1/conversations/{id}/messages 
RUN_ID=$(
  curl -s http://127.0.0.1:8080/v1/runs \
    -H 'content-type: application/json' \
    -d '{
      "prompt": "Find the risky code path",
      "workspace_path": "'$PWD'",
      "model": "deepseek-v4-pro",
      "max_steps": 8
    }' | jq -r .run_id
)

curl -N "http://127.0.0.1:8080/v1/runs/$RUN_ID/events"
